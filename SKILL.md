# 银行流水财务分析报告生成器

输入一份银行流水（xlsx/csv，列名随意），一键产出一套专业财务分析文件。脚本负责算数与排版，**解读文字由你（Claude）读完数据后现写**，保证贴合这份流水的实际情况。全自动执行，无需中途确认。

## 资产脚本（已就绪，勿改路径）

- `~/.claude/skills-assets/bank-flow-report/flow_analyze.py` —— 配置驱动的分析引擎，输出 `analysis/data.json`
- `~/.claude/skills-assets/bank-flow-report/flow_build.py` —— 渲染引擎，输出三件套 HTML + `pic/` 配图位

## 工作流程

### Step 0：确认输入
确认用户提供的是银行账户流水（含日期、金额、收/支方向或正负、对手方等列）。若不是流水数据，提示后终止，不生成文件。

### Step 1：读流水 → 写 config.json
1. **工作目录** = 流水文件所在目录。若该目录混有无关文件，新建子目录 `流水分析_<主体简称>/` 并把流水复制进去，在其中工作。
2. 用 pandas 读取：列出全部 **sheet 名**与**列名**，并打印前 5 行，理解数据结构。
3. **识别列**，映射到标准字段（按下表的常见同义词判断）：

   | 标准字段 | 含义 | 常见列名 |
   |---|---|---|
   | `date` | 交易时间 | 实际发生时间、交易时间、交易日期、记账日期、日期 |
   | `amount` | 发生额 | 发生额、交易金额、金额、发生金额 |
   | `dc` | 借贷标志 | 借贷标志、收付标志、借贷、方向、资金收支 |
   | `counterparty` | 对手方 | 对方户名、对手方、交易对手、对方名称、收付方 |
   | `summary` | 摘要 | 摘要、交易摘要、用途 |
   | `memo` | 备注 | 交易备注、备注、附言、用途说明 |
   | `balance` | 余额 | 余额、账户余额、交易后余额 |

4. 判断 `dc` 的**收入值与支出值**（如 贷/借、收/付、进/出）。填入 `dc_in` / `dc_out`。
5. **特殊格式**：若没有统一的"金额+方向"列，而是 ① 金额带正负，或 ② 分"收入额/支出额"两列 —— 先写个小脚本把流水转成标准列（新增"发生额"取绝对值、"借贷标志"按正负或来源列填），存成临时 xlsx，再继续。
6. 提取**主体信息**并**判断主体类型**（决定 Step 3 的解读视角）：
   - 账户名/对手方含"协会、促进会、基金会、商会、校友会、学会" → `非营利/社团`
   - 含"公司、有限、集团、厂、店" → `企业`
   - 是个人姓名 → `个人`
7. 多 sheet 时设 `dedup_priority`（更完整/更新的 sheet 名靠前，优先保留）。
8. 在工作目录写 `config.json`：

```json
{
  "src": "流水文件名.xlsx",
  "outdir": ".",
  "cols": {"date":"实际发生时间","amount":"发生额","dc":"借贷标志","counterparty":"对方户名","summary":"摘要","memo":"交易备注","balance":"余额"},
  "dc_in": "贷", "dc_out": "借",
  "dedup_priority": ["新数据","老数据"],
  "dedup": "auto",
  "big_threshold": 200000,
  "mask_site": true,
  "subtitle": "银行账户流水财务分析报告",
  "report_date": "今天日期 YYYY-MM-DD",
  "meta": {"name":"主体全称","account":"账号","subject_type":"非营利/社团"}
}
```
说明：`big_threshold` 大额阈值按规模酌定（社团 20 万、个人可设 5 万）；`mask_site:true` 时网站与图表版脱敏（保留姓、名转 `*`，如 `张伟明→张**`）、文字版留真名（用户既定偏好）；`dedup` 默认 `"auto"`（仅多 sheet 时按 `dedup_priority` 去重；单 sheet 不去重，避免误删同日同额交易），可填 `true`/`false` 强制；CSV 编码自动探测（utf-8/gbk/gb18030），异常时用 `encoding` 字段指定。

### Step 2：跑分析引擎
```
python3 ~/.claude/skills-assets/bank-flow-report/flow_analyze.py <工作目录>/config.json
```
检查打印的收入/支出/记录数是否合理。若提示"未解析到记录"，回到 Step 1 核对列映射。

### Step 3：读 data.json → 写 narrative.json（核心：你来写解读）
读 `analysis/data.json`，理解这份流水的真实情况，**按主体类型切换视角**撰写解读，写到 `analysis/narrative.json`。

**视角差异：**
- **非营利/社团**：以收定支、政府/财政依赖度、最低储备线、资金跑道、个人收款合规
- **企业**：盈利与净现金流、回款节奏与账期、成本结构、客户/供应商集中度
- **个人**：收支平衡、储蓄率、大额与异常支出、现金流稳定性

**narrative.json 结构**（字段可含 `<b>`、`<span class='red'>`、`<span class='num'>` 等行内标签）：
```json
{
  "exec_summary": "执行摘要，一段话概述规模与核心判断",
  "alert": {"title":"最关键风险的标题", "body":"风险描述（无突出风险可省略整个 alert 字段）"},
  "findings": [{"t":"发现标题","d":"一句话说明"}, "...6 条左右"],
  "overview_note": "资金总览解读",
  "income_note": "收入结构解读",
  "expense_note": "支出结构解读",
  "trend": [{"t":"阶段/拐点标题","d":"说明"}, "...2-4 条"],
  "trend_note": "季节性或趋势补充（显示为小字注释）",
  "risk_note": "风险与异常审视的结论",
  "conclusion": "综合结论，一段话",
  "advice": [{"tag":"紧急","h":"建议标题","p":"建议内容"}, "...4-6 条"]
}
```
`advice.tag` 用 紧急/重要/战略/机制/合规，会渲染成对应颜色。数字与表格无需在此重复，引擎自动从 data.json 生成。

### Step 4：生成三件套
```
python3 ~/.claude/skills-assets/bank-flow-report/flow_build.py <工作目录>/config.json
```
产出工作目录下：`index.html`（整合网站，主成品）、`charts.html`（图表版）、`report.html`（完整文字版，留真名存档）、`pic/`（配图位）。

### Step 5：打开并交付配图清单
1. `open <工作目录>/index.html`
2. 向用户说明：三件套已生成、脱敏策略、以及**配图清单**——把图片放进 `pic/`，文件名固定如下，全部可选，缺图自动显示品牌渐变不破版：

   | 文件名 | 位置 | 建议内容 | 尺寸 |
   |---|---|---|---|
   | `hero.jpg` | 首屏大背景 | 城市/行业相关、深色调 | 1920×1080 |
   | `fund.jpg` | 资金总览横幅 | 金融/数据/报表 | 1600×500 |
   | `income.jpg` | 收入结构横幅 | 合作/资金注入 | 1600×500 |
   | `expense.jpg` | 支出结构横幅 | 团队/协作 | 1600×500 |
   | `trend.jpg` | 趋势横幅 | 曲线/时间/道路 | 1600×500 |
   | `risk.jpg` | 风险横幅 | 审计/盾牌 | 1600×500 |
   | `advice.jpg` | 建议横幅 | 灯塔/航向 | 1600×500 |
   | `logo.png` | 导航栏 | 主体标志，透明背景 | 高 ≥56px |

## 注意事项
- 全自动执行，不中途追问；列映射或主体类型不确定时取最合理判断并在最终说明里告知。
- 解读必须基于 data.json 的真实数字，不杜撰；金额一律用"万元/亿元"口径，与引擎一致。
- 不修改用户原始流水文件；脱敏只作用于产出的网站与图表版。
- 本报告基于银行流水单一信息源，结论里需保留"不构成正式审计意见"的免责表述。
